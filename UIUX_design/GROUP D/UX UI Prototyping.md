# UX/UI Prototyping

## Purpose of the prototype
The purpose of my prototype is exploration and communication.
I want to explore how the core interactions of the RDI platform work and communicate the concept clearly to users before implementation.

## Fidelity level
I chose a low‑fidelity prototype because my project is still in the early stage.
Low‑fidelity wireframes help me focus on:
structure,flow,user tasks
without spending time on visual details.

## Key user tasks prototyped
I prototyped the following core tasks:
A company searches for suitable students
A student creates a project‑ready profile
A teacher/admin posts an announcement
These tasks represent the main value of the platform.

## Testing the prototype
I showed the prototype to users and asked them to perform realistic tasks, such as:
“Find a student with AI skills.”
“Add skills to your student profile.”
“Check the latest announcement.”
I observed silently while they interacted with the prototype.

## Documented findings
Usability issues
Some users did not understand how keyword search works.
Students were unsure which skills to add.
Companies expected filters.
Misunderstandings
Some students thought the system would automatically recommend them to companies.
Companies expected “best match” sorting, which the prototype did not include.
User interpretations
Users liked the simple layout.
They understood the three user roles clearly.
They expected more guidance when creating a profile.

## Iteration based on feedback
Based on the findings, I improved the prototype by:
Adding hints under the search bar
Adding a “profile completeness” indicator
Adding optional filters for companies
Making required fields more visible
Clarifying that matching is based on keywords
These changes help align the system with user expectations and reduce confusion.
